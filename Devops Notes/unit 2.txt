
DOCKER COMPLETE DETAILED NOTES

1. IMAGE LAYERING
- Every Dockerfile instruction creates a layer
- Layers are cached → improves performance
- Only changed layers are rebuilt

Example:
FROM ubuntu:22.04
RUN apt update
RUN apt install -y nginx

2. BUILD CONTEXT & .dockerignore
- Build context = folder sent to Docker daemon
Command:
docker build -t app .

.dockerignore:
node_modules
.git
.env
__pycache__

3. DOCKERFILE INSTRUCTIONS (DETAILED)

FROM → base image
RUN → executes command
COPY → copies local files
ADD → supports URL + auto extract
CMD → default command
ENTRYPOINT → fixed command
WORKDIR → working directory
ENV → env variables
EXPOSE → documentation of ports
VOLUME → persistent storage

Example Full Dockerfile:
FROM node:18
WORKDIR /app
COPY package.json .
RUN npm install
COPY . .
EXPOSE 3000
CMD ["node","app.js"]

4. DOCKER BUILD PROCESS

Step-by-step:
1. Read Dockerfile
2. Send build context
3. Execute instructions sequentially
4. Cache layers
5. Final image created

Commands:
docker build -t myapp:v1 .
docker build --no-cache -t myapp:v2 .

5. IMAGE TAGGING & VERSIONING

Format:
repository:tag

Examples:
docker tag myapp:v1 myrepo/myapp:latest
docker images

Best Practices:
- latest (unstable)
- v1.0, v1.1 (stable)

6. IMAGE INSPECTION

docker inspect image_id
docker history image_id

7. DOCKER NETWORKING (DETAILED)

Bridge:
docker network create my_bridge
docker run -d --network my_bridge nginx

Host:
docker run -d --network host nginx

Overlay:
docker network create -d overlay my_overlay

List networks:
docker network ls

Inspect network:
docker network inspect my_bridge

8. DNS IN DOCKER
- Containers resolve names via internal DNS

Example:
ping container_name

9. LINKING CONTAINERS

docker run -d --name db mysql
docker run -d --name app --link db nginx

10. PORT MAPPING

docker run -p 8080:80 nginx
docker run -p 127.0.0.1:5000:5000 app

11. STORAGE (DETAILED)

Volumes:
docker volume create vol1
docker run -v vol1:/data nginx

Bind Mount:
docker run -v /home/user/data:/data nginx

Inspect volume:
docker volume inspect vol1

12. COPY ON WRITE

- Base layers read-only
- Container adds writable layer
- Changes stored separately

13. REGISTRIES

Docker Hub:
docker login
docker push username/app

GHCR:
docker login ghcr.io

Private Registry:
docker run -d -p 5000:5000 registry

Push to private:
docker tag app localhost:5000/app
docker push localhost:5000/app

14. AUTHENTICATION

docker login
docker logout

Using Token:
docker login -u username
(password = token)

END
